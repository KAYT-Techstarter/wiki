1. Was sind Backups?

Ein Backup ist eine Sicherungskopie von Dateien oder ganzen Systemen, um Datenverluste zu vermeiden. Diese Sicherung kann auf verschiedenen Medien wie Festplatten, Cloud-Speicher oder externen Datenträgern gespeichert werden.

2. Warum sollte man Backups machen?

Backups sind wichtig, weil sie vor ungewolltem Datenverlust schützen, der durch Hardware-Fehler, Hackerangriffe oder menschliche Fehler entstehen kann. Ein regelmäßiges Backup hilft, verlorene Daten schnell wiederherzustellen.

3. Was macht der backup.py Code?

Hier ist der gegebene Code als Codeblock:

import shutil
import datetime

backup_source = "C:/Users/benutzer/Dokumente"
backup_target = "D:/Backup_" + datetime.datetime.now().strftime("%Y%m%d%H%M")

shutil.copytree(backup_source, backup_target)
print(f"Backup erfolgreich: {backup_target}")

Dieser Code macht Folgendes:

  1. Importiert die Module shutil und datetime, die für Dateioperationen und Zeitstempel benötigt werden.
  2. Definiert die Quell- und Zielverzeichnisse für das Backup.
  3. Das Zielverzeichnis wird mit einem Zeitstempel im Format YYYYMMDDHHMM erstellt.
  4. Der gesamte Inhalt des Quellverzeichnisses wird in das Backup-Verzeichnis kopiert.
  5. Nach erfolgreicher Sicherung wird eine Bestätigungsmeldung mit dem Speicherort des Backups ausgegeben.

4. Automatisches Ausführen eines Scripts

Ja, es gibt verschiedene Möglichkeiten, ein Script automatisch auszuführen:

  • Unter Windows kann man die Aufgabenplanung nutzen.

Durch die automatische Ausführung eines Scripts um 2 Uhr morgens wird sichergestellt, dass Backups regelmäßig erstellt werden, ohne dass man sie manuell starten muss.

Alternativ kann man mit Windows + R und shell:startup das Script in den Autostart-Ordner legen. Dadurch wird beim Systemstart ein Backup erstellt, allerdings nicht zwingend um 2 Uhr morgens.
